## Responsive Breakpoint Contract

**New for Phase 6.** Defines the responsive behavior contract across all 7 pages.

### Breakpoint Tokens

| Breakpoint | Width | Tailwind Prefix | Purpose |
|------------|-------|-----------------|---------|
| mobile | 320px–767px | *(default)* | Single-column layouts, stacked content, hamburger menu |
| tablet | 768px–1279px | `md:` | Two-column grids, expanded navigation visible |
| desktop | 1280px+ | `lg:` | Three-column grids, full-width layouts at max container |

### Responsive Behavior Contracts

#### 1. Navigation
- **Mobile (<768px):** Hamburger menu visible, desktop nav links hidden (`hidden md:flex`). Mobile menu slides open with `grid-template-rows: 0fr→1fr` animation over 300ms.
- **Tablet+ (≥768px):** Desktop nav links visible, hamburger hidden. "Minta Penawaran" CTA button visible.

#### 2. Grid Layouts
- **Product bento grid (home):** `grid-cols-1` → `md:grid-cols-2` → `lg:grid-cols-3`
- **About page history:** `grid-cols-1` → `md:grid-cols-2`
- **Contact page:** Info panel + form stack vertically on mobile, side-by-side on `md:`
- **Subsidiary cards (about):** `grid-cols-1` → `md:grid-cols-3`
- **Clinical applications grid (product pages):** `grid-cols-1` → `md:grid-cols-2` → `lg:grid-cols-3` (consumables: 4-column → `md:grid-cols-2` → `lg:grid-cols-4`)

#### 3. Typography Scaling
- `headline-xl`: `text-[28px]` mobile → `md:text-[48px]` desktop
- `headline-lg`: `text-[28px]` mobile → `text-[32px]` desktop (inline class, not token)
- Stat numbers: `text-[36px]` mobile → `text-[48px]` desktop

#### 4. Touch Targets
- All interactive elements (links, buttons, form inputs) must have minimum `44px × 44px` touch target on mobile.
- Mobile menu items: `min-h-[44px]` (already implemented).
- Form inputs: `py-3` + `px-4` yields ~48px height (already compliant).

#### 5. Spec Table (product pages)
- **Mobile (<768px):** `<thead>` hidden, each `<tr>` renders as `flex flex-col` with label first (already implemented via `@media (max-width: 767px)` in `main.css`).
- **Tablet+:** Standard table layout.

#### 6. Floating WhatsApp Button
- **Desktop:** `56px × 56px`, `bottom: 24px; right: 24px`
- **Mobile (<768px):** `48px × 48px`, `bottom: 16px; right: 16px` (already implemented)

#### 7. Container Padding
- **Mobile:** `px-[--spacing-margin-mobile]` (16px)
- **Tablet+:** `md:px-[--spacing-margin-desktop]` (48px)

## Page Transition Contract

**New for Phase 6.** Defines the page transition behavior for this multi-page static site (Vite MPA, no SPA routing).

### Transition Type
- **Fade transition** on page load: content fades in from `opacity: 0` to `opacity: 1` over 300ms.
- No fade-out on page unload (browser handles navigation; fade-out would block perceived performance).

### Implementation
- All `<main id="main-content">` elements receive initial CSS: `opacity: 0; animation: pageFadeIn 300ms ease-out forwards;`
- `@keyframes pageFadeIn { from { opacity: 0; } to { opacity: 1; } }`
- Applied via a single CSS class `.page-enter` added to `<main>` on every page.

### Timing & Easing
- **Duration:** 300ms
- **Easing:** `ease-out` (starts fast, decelerates — feels natural for content appearing)
- **Delay:** 0ms (starts immediately on DOM ready)

### Reduced Motion
- When `prefers-reduced-motion: reduce` is active, `.page-enter` must have `animation: none; opacity: 1;` (content visible immediately, no animation).
- Already handled by the existing `@media (prefers-reduced-motion: reduce)` block in `main.css` which sets all animation/transition durations to `0.01ms`.

### Scroll Behavior
- On page load, scroll position must be at top (`window.scrollTo(0, 0)`).
- Smooth scroll for in-page anchor links already implemented via `home.js` (`.home-scroll` class). Extend to all pages: `html { scroll-behavior: smooth; }` in `main.css`.
- Reduced motion: `scroll-behavior: auto;` when `prefers-reduced-motion: reduce`.

## SEO Meta Tag Contract

**New for Phase 6.** Every page must have unique, complete meta tags. Currently only `<title>` and `<meta name="description">` exist on some pages. No Open Graph or canonical tags exist.

### Required Tags Per Page

Every page must include in `<head>`:

```html
<meta charset="UTF-8">
<meta name="viewport" content="width=device-width, initial-scale=1.0">
<title>{page-specific title}</title>
<meta name="description" content="{page-specific description, 120-160 characters}">
<link rel="canonical" href="{full URL}">

<!-- Open Graph -->
<meta property="og:title" content="{same as <title>}">
<meta property="og:description" content="{same as meta description}">
<meta property="og:type" content="website">
<meta property="og:site_name" content="APBA Group">
<meta property="og:locale" content="id_ID">
<meta property="og:image" content="{full URL to representative image, 1200x630px minimum}">
<meta property="og:url" content="{same as canonical}">

<!-- Twitter Card -->
<meta name="twitter:card" content="summary_large_image">
<meta name="twitter:title" content="{same as <title>}">
<meta name="twitter:description" content="{same as meta description}">
<meta name="twitter:image" content="{same as og:image}">
```
