Test structure using `describe.each` for parameterized tests across all 4 products:

**Test Group 1: Page Structure (per product, 4 products × 6 checks = 24 tests)**
For each product page, verify:
1. File exists in `src/products/`
2. Contains `<main id="main-content">` element
3. Contains breadcrumb with "Produk > [Product Name]"
4. Contains product name in `<h1>` or hero heading
5. Contains "Spesifikasi Teknis" section heading
6. Contains "Aplikasi Klinis" section heading

**Test Group 2: Specifications Table (per product, 4 products × 3 checks = 12 tests)**
For each product page, verify:
1. Contains `<table>` element with `<thead>` and `<tbody>`
2. Contains at least 6 data rows in the table
3. Table has 2 columns (spec name and value)

**Test Group 3: Clinical Applications (per product, 4 products × 2 checks = 8 tests)**
For each product page, verify:
1. Contains at least 3 application cards (count elements with application card pattern)
2. Each card has an icon (Material Symbols span), title, and description

**Test Group 4: Purchase/Rental Banner (4 tests)**
- Blue Diode Laser: contains "Tersedia untuk Pembelian atau Penyewaan"
- Ultrasonic Surgery: contains "Tersedia untuk Pembelian atau Penyewaan"
- Laser Holmium: contains "Tersedia untuk Pembelian atau Penyewaan"
- Consumable Urologi: does NOT contain "Tersedia untuk Pembelian atau Penyewaan"

**Test Group 5: WhatsApp CTA (per product, 4 products × 3 checks = 12 tests)**
For each product page, verify:
1. Contains link to `wa.me/6281112129779`
2. Link contains URL-encoded product name
3. Link has `target="_blank"` and `rel="noopener noreferrer"`

**Test Group 6: Email CTA (per product, 4 products × 3 checks = 12 tests)**
For each product page, verify:
1. Contains `mailto:info@apbagroup.com` link
2. Link contains URL-encoded product name in subject
3. Link contains pre-filled body template with "Nama:" and "Institusi:" fields

**Test Group 7: Related Products (per product, 4 products × 4 checks = 16 tests)**
For each product page, verify:
1. Contains "Produk Lainnya" section heading
2. Contains exactly 3 related product cards
3. Related product links do NOT include the current page's URL (no self-link)
4. Each related product link points to a valid product page URL pattern (`/products/*`)

**Test Group 8: Accessibility (per product, 4 products × 3 checks = 12 tests)**
For each product page, verify:
1. Product hero image has descriptive alt text (not empty)
2. All interactive elements have `min-h-[44px]` or equivalent touch target
3. Page has `<html lang="id">` attribute

Total: ~100 test cases.

Use `fs.readFileSync` to read HTML files, then use regex or simple string matching for verification. Follow the same test file patterns as `tests/accessibility.test.js`.
