# Phase 3 Plan: Home Page

**Goal:** Visitors arriving at the home page immediately understand APBA Group's value proposition and can navigate to any product or section

**Mode:** mvp

**Depends on:** Phase 2 (shared layout & navigation)

**Requirements:** HOME-01, HOME-02, HOME-03, HOME-04, HOME-05, HOME-06, HOME-07, HOME-08

---

## Context

- `home.html` currently contains a design token verification page — needs full replacement
- Header (`src/partials/header.html`) and footer (`src/partials/footer.html`) are already built and injected by the build pipeline
- `nav.js` handles mobile menu, active states, and escape key
- CSS tokens in `src/css/main.css` are complete (50+ colors, typography, spacing)
- 4 local product images exist in `public/images/` (WebP + PNG fallbacks)
- DESIGN.MD contains the full home page reference layout with all sections
- All content in Indonesian (Bahasa Indonesia)
- Vite multi-page build: `src/home.html` → `dist/index.html` (root route)

## Approach

Replace `home.html` content (between header and footer) with 6 sections matching DESIGN.MD:
1. **Hero** — full-viewport gradient overlay on corporate office image, company name, tagline, 2 CTAs
2. **About Summary** — 2-column layout with office image + text, founding year (2017) + hospital count (270+)
3. **Products & Services Bento Grid** — 6 cards (3 product cards with images, 3 service cards with icons)
4. **Why Choose Us** — 5 feature cards in 3-column grid
5. **Subsidiaries** — 3 glassmorphism cards on deep blue background
6. **Stats** — animated counters (2017, 270+) that count up on scroll

New JS: `home.js` for smooth scroll + intersection observer for animated counters.

### Task 1.2: Create home.js for smooth scroll + animated counters

**Files created:** `src/js/home.js`

1. **Smooth Scroll** (HOME-08)
   - Intercept clicks on anchor links within the home page
   - `scrollIntoView({ behavior: 'smooth', block: 'start' })`
   - Account for fixed header offset (80px) by adjusting scroll position

2. **Animated Counters** (HOME-07)
   - IntersectionObserver to detect when stats section enters viewport
   - Count up from 0 to `data-target` value over ~2s duration
   - Support "+" suffix (e.g., "270+")
   - Support dynamic calculation (years since 2017)
   - Only animate once (unobserve after animation)
   - Respect `prefers-reduced-motion` (skip animation, show final value immediately)

**Files changed:** `src/home.html` — add `<script type="module" src="/src/js/home.js"></script>` before closing `</body>`

## Threat Model

| Threat | Mitigation |
|--------|-----------|
| External image URLs in DESIGN.MD don't work | Use local images from `public/images/` only |
| Animated counters flash wrong values | Start at 0, animate up; reduced-motion shows final value |
| Smooth scroll conflicts with fixed header | Offset scroll position by 80px (header height) |
| home.js loads on pages that don't need it | Only include in home.html, not in shared partials |
| Product card links break if routes change | Links match vite.config.js input keys exactly |
