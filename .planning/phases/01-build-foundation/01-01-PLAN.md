<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| developer→build system | Developer runs npm commands that execute code from node_modules |
| build system→Vercel | Build output deployed to Vercel edge network |
| test runner→file system | Vitest tests read dist/ directory to verify build output |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-01-01 | Spoofing | npm package installation | mitigate | All packages in Standard Stack verified legitimate via npm registry (RESEARCH.md Package Legitimacy Audit). Run `npm audit` before phase completion. Use package-lock.json for deterministic installs. |
| T-01-02 | Tampering | build time code injection | mitigate | Only install packages from official npm registry. No postinstall scripts in dependencies. Vite's static build prevents runtime injection. |
| T-01-03 | Information Disclosure | build output in dist/ | accept | Dist folder contains static HTML/CSS only. No sensitive data in Phase 1 scope. No API keys or secrets. |
| T-01-04 | Denial of Service | build failure | accept | Local development only. Build failures are caught by `npm run build` command. No production availability impact in Phase 1. |
| T-01-SC | Tampering | npm/pip installs | mitigate | All packages [OK] per slopcheck (RESEARCH.md line 90). No [SUS] or [SLOP] packages. No human checkpoint needed. |
</threat_model>
