# Phase 5 — UI Design Contract

> Visual and interaction contract for frontend phases. Generated by gsd-ui-researcher, verified by gsd-ui-checker.

---

## Design System

| Property | Value |
|----------|-------|
| Tool | none (Vite MPA + Tailwind CSS v4, manual tokens) |
| Preset | not applicable |
| Component library | none (vanilla HTML, shared header/footer duplicated across pages) |
| Icon library | Material Symbols Outlined (Google Fonts CDN) |
| Font | Plus Jakarta Sans (headlines, 600/700), Inter (body, 400/600) |

**Source:** Pre-populated from `src/css/main.css` (Phase 1 tokens) and `DESIGN.MD`. No `components.json` exists — this is a Vite MPA with 7 static HTML pages, not a React project. Consistent with Phase 4 UI-SPEC.

## Spacing Scale

Declared values (multiples of 4 only, from existing `@theme` tokens in `src/css/main.css`):

| Token | Value | Usage |
|-------|-------|-------|
| stack-sm | 8px | Icon-to-text gaps, inline padding, form label-to-input gaps |
| stack-md | 16px | Default element spacing, card internal padding, form field gaps |
| gutter | 24px | Grid column gaps, horizontal spacing between related elements |
| stack-lg | 32px | Section-to-section vertical spacing within page content |
| section-padding | 80px | Major section top/bottom padding (hero, history, mission, certifications, subsidiaries, contact form) |
| margin-mobile | 16px | Horizontal page margin on mobile (<768px) |
| margin-desktop | 48px | Horizontal page margin on desktop (>=768px) |
| container-max | 1280px | Max content width for all page sections |

Exceptions:
- Touch targets: minimum 44px height for all interactive elements (form inputs, buttons, WhatsApp floating button, links). Already enforced in Phase 2 shared layout.
- Form inputs: padding `py-3 px-4` (12px/16px) for comfortable typing area — not a multiple of 4 but standard for form UX.
- Floating WhatsApp button: 56px diameter (fixed size, not from spacing scale).

## Color

| Role | Value | Usage |
|------|-------|-------|
| Dominant (60%) | `#f9f9ff` (surface) | Page background, content area surfaces |
| Secondary (30%) | `#ffffff` (surface-container-lowest) | Cards, form container, certification badges, subsidiary cards |
| Accent (10%) | `#00A3AD` (medical-teal) | Section divider bars, icon accents, active states, hover states, WhatsApp floating button, form focus rings |
| Deep Blue | `#003366` (deep-blue) | Hero backgrounds, headings, footer background, submit button |
| Destructive | `#ba1a1a` (error) | Form validation error states (border color, error text) |
| Alert Gold | `#FFB800` (alert-gold) | Certification badge icon background (ISO 13485 prominence indicator) |
| Surface Variant | `#d8e3fb` (surface-variant) | Hero subtitle text, secondary info text |
| On Surface Variant | `#43474f` (on-surface-variant) | Body text in light sections, form placeholder text |
| Border Light | `#E2E8F0` (border-light) | Card borders, form input borders, section dividers |

**60/30/10 split applied per page:**
- 60% surface (`#f9f9ff`) — page background between sections
- 30% white cards (`#ffffff`) — history cards, mission/value cards, certification badges, subsidiary cards, form container
- 10% medical-teal (`#00A3AD`) — section accent bars, icon highlights, link hovers, floating WhatsApp button, form focus rings

**Accent (`#00A3AD`) reserved for:**
1. Section divider accent bars (32px wide, 4px tall, above section headings)
2. Icon color in mission/value cards, certification badges, contact info icons
3. Form input focus ring (`focus-visible:ring-2 focus-visible:ring-medical-teal`)
4. Link hover states
5. Active navigation state (already defined in Phase 2)
6. Floating WhatsApp button background
7. Submit button hover state

## Interaction Contract

| Interaction | Behavior |
|-------------|----------|
| Form submit (display-only) | Show success toast/modal: "Pesan Anda telah tercatat. Tim kami akan menghubungi Anda segera." No actual submission — form has no backend. |
| Form validation error | Invalid field gets `border-error` (`#ba1a1a`) red border, error message below field in `caption` size, `text-error` |
| Form input focus | `focus:ring-2 focus:ring-medical-teal focus:border-transparent` — teal ring, transparent border |
| WhatsApp floating button click | Opens `wa.me` in new tab with pre-filled message |
| WhatsApp link in contact info click | Opens `wa.me` in new tab |
| Email link click | Opens default mail client with `mailto:info@apbagroup.com` |
| Phone link click | Initiates call on mobile devices (`tel:081112129779`) |
| CTA banner WhatsApp click | Opens `wa.me` in new tab with consultation pre-filled message |
| Card hover (mission, certification, subsidiary) | Shadow increases to `shadow-[0px_4px_20px_rgba(0,51,102,0.05)]` |
| Subsidiary glass card hover | Background opacity increases from `bg-white/10` to `bg-white/15` |
| Submit button hover | Background changes from `bg-deep-blue` to `bg-medical-teal` |
| All CTAs | Min 44px touch target, visible focus ring (`focus-visible:ring-2 focus-visible:ring-medical-teal`) |
| Floating WhatsApp button hover | Background changes from `bg-medical-teal` to `bg-white`, icon changes to `text-medical-teal`, shadow increases |
| Reduced motion | All animations disabled per existing `main.css` `prefers-reduced-motion` media query |

## Accessibility Contract

| Requirement | Implementation |
|-------------|---------------|
| Skip link | Already present in Phase 2 shared header |
| Form labels | All inputs have visible `<label>` elements with `for` attribute matching input `id` |
| Required fields | `required` attribute on 5 of 6 form fields (product interest is optional) |
| Focus visible | All interactive elements: `focus-visible:ring-2 focus-visible:ring-medical-teal focus-visible:ring-offset-2` |
| Color contrast | All text meets WCAG AA 4.5:1 minimum (deep-blue on white = 12.6:1, medical-teal on white = 3.2:1 — use medical-teal only for large text/icons or on dark backgrounds) |
| Semantic HTML | `<main>`, `<section>`, `<h1>`–`<h3>` hierarchy, `<form>` with proper structure, `<label>` for all inputs |
| ARIA labels | Floating WhatsApp button: `aria-label="Chat via WhatsApp"`, form inputs with descriptive labels |
| Reduced motion | Already handled in Phase 1 `main.css` via `prefers-reduced-motion` media query; floating button pulse animation respects this |
| Touch targets | All interactive elements minimum 44px height (form inputs, buttons, links, floating button 48px on mobile) |
